4.2 P/BV Ratio Analysis

📘 P/BV Analysis — ราคาต่อมูลค่าทางบัญชี
P/BV Analysis P/BV ของ STPI: 0.82 เท่า (ต่ำกว่า 1 = ซื้อได้ต่ำกว่า Book Value) BPS: 4.83 บาท ROE: 2.51% (ต่ำกว่า Cost of Equity ~10%) P/BV ที่เหมาะสม (Justified P/BV): P/BV = (ROE - g) / (r - g) สมมติ: ROE = 2.51%, g = 2%, r = 9.5% P/BV = (0.0251 - 0.02) / (0.095 - 0.02) = 0.0051 / 0.075 = 0.07 เท่า (ต่ำมาก!) → ถ้าใช้ ROE ปัจจุบัน P/BV ควรต่ำมาก → แต่ตลาดให้ P/BV 0.82 → ตลาดคาดว่า ROE จะฟื้นตัว

Target Price ถ้า ROE ฟื้นตัว

roe recovery scenarios ถ้า ROE ฟื้นเป็น 10%: P/BV = (0.10 - 0.02) / (0.095 - 0.02) = 1.07 เท่า Target = BPS 4.83 × 1.07 = 5.17 บาท ถ้า ROE ฟื้นเป็น 15%: P/BV = (0.15 - 0.02) / (0.095 - 0.02) = 1.73 เท่า Target = BPS 4.83 × 1.73 = 8.36 บาท

5 Absolute Valuation — DCF (Discounted Cash Flow)

DCF เป็นวิธีประเมินมูลค่าที่แม่นยำที่สุดในทางทฤษฎี โดยคำนวณมูลค่าปัจจุบันของกระแสเงินสดอิสระที่บริษัทจะสร้างได้ในอนาคต

📐 ขั้นตอนการทำ DCF
DCF Steps ขั้นตอนที่ 1: คำนวณ WACC (Weighted Average Cost of Capital) ขั้นตอนที่ 2: คาดการณ์ FCFF 5 ปี (Free Cash Flow to Firm) ขั้นตอนที่ 3: คำนวณ Terminal Value ขั้นตอนที่ 4: คิดลดกลับมาเป็นมูลค่าปัจจุบัน ขั้นตอนที่ 5: Sensitivity Analysis

ขั้นตอนที่ 1: คำนวณ WACC

WACC Calculation ═══ Cost of Equity (Re) — CAPM ═══ Rf (Risk-Free Rate) = 2.8% (พันธบัตรรัฐบาลไทย 10 ปี) Beta = 1.05 ERP (Equity Risk Premium) = 7.5% (Mature ERP 5.5% + Country Risk 2.0%) Re = 2.8% + 1.05 × 7.5% = 2.8% + 7.875% = 10.68% ═══ Cost of Debt (Rd) ═══ Rd (ประมาณการ) = 5.5% Tax Rate (Effective) = 15.69% ═══ โครงสร้างเงินทุน ═══ Market Cap (E) = 7,540M Debt (D) = 3,660M V = E + D = 11,200M E/V = 67.3% D/V = 32.7% ═══ WACC ═══ WACC = (67.3% × 10.68%) + (32.7% × 5.5% × (1 - 15.69%)) = 7.19% + 1.52% = 8.71% → ใช้ WACC = 9.0% (ปัดขึ้นเพื่อ Conservative)

6 EPV — Earnings Power Value

EPV ประเมินมูลค่าจาก "กำลังทำกำไรปัจจุบัน" โดยไม่สมมติการเติบโต — ถ้า EPV สูงกว่าราคาตลาด แสดงว่าหุ้นถูกแม้ไม่เติบโต

EPV Calculation Normalized EBIT (TTM): 297M อัตราภาษี: 16% WACC: 9.0% NOPAT = 297 × (1 - 0.16) = 249.5M EPV = NOPAT / WACC = 249.5 / 0.09 = 2,772M Equity Value = EPV - Debt + Cash = 2,772 - 3,660 + 520 = -368M ❌ → EPV ติดลบ! กำไรปัจจุบันไม่เพียงพอรองรับมูลค่าหนี้สิน
ถ้า EBIT ดีขึ้น สมมติ EBIT ปรับตัวเป็น 500M: EPV = 500 × 0.84 / 0.09 = 420 / 0.09 = 4,667M Equity Value = 4,667 - 3,660 + 520 = 1,527M มูลค่าต่อหุ้น = 1,527 / 1,812 = 0.84 บาท → แม้ EBIT ดีขึ้นเป็น 500M EPV ก็ยังต่ำกว่าราคาตลาดมาก → ตลาดจ่ายเพื่อ "การเติบโต" ไม่ใช่ "กำลังทำกำไรปัจจุบัน"
❌ EPV บอกอะไร?
ถ้าบริษัทไม่เติบโตเลย มูลค่าจะต่ำกว่าราคาตลาดมาก — นักลงทุนที่ซื้อ STPI กำลังจ่ายเพื่อ "ความคาดหวังการเติบโต" ซึ่งมีความเสี่ยงถ้าการเติบโตไม่เกิดขึ้น

11 Catalyst & Risk Monitoring

🟢 Catalyst เชิงบวก

  • Backlog ใหม่มูลค่าสูง → รายได้อนาคตชัดเจนขึ้น
  • EBIT Margin ฟื้นตัวเกิน 12% → ROE จะดีขึ้นมาก
  • ลดหนี้สินอย่างมีนัยสำคัญ → ลดความเสี่ยงทางการเงิน
  • เริ่มจ่ายเงินปันผล → ดึงดูดนักลงทุนรายได้
  • โครงการ Energy Transition / LNG → โอกาสงานใหม่

🔴 Catalyst เชิงลบ

  • Backlog ลดลงต่อเนื่อง → รายได้จะหดตัว
  • Interest Coverage ลดต่ำกว่า 1.0 → เสี่ยงผิดนัดชำระหนี้
  • เพิ่มทุนอีก → Dilution เพิ่ม
  • ราคาเหล็กพุ่งสูง → กดดัน Margin
  • คู่แข่งจากจีน/เวียดนามตัดราคา → สูญเสียส่วนแบ่งตลาด

12 Checklist การ Valuation

ใช้ Checklist นี้เพื่อตรวจสอบว่าการวิเคราะห์ครบถ้วนหรือไม่ — คลิกที่ช่องเพื่อเปลี่ยนสถานะ

✅ ก่อนเริ่ม Valuation

  • ✓ เข้าใจธุรกิจ: รับเหมาผลิตโครงสร้างเหล็ก, หม้อไอน้ำ, Piping
  • ✓ ดูงบการเงินย้อนหลัง (รายได้ 5 ปี + TTM)
  • ✓ รู้จักกลุ่มอุตสาหกรรม: Basic Materials / Steel Fabrication
  • ☐ อ่าน 56-1 One Report ฉบับล่าสุด (แนะนำให้อ่านเพิ่มเติม)

✅ ระหว่าง Valuation

  • ✓ ใช้ 7 วิธีในการประเมินมูลค่า (P/E, P/BV, EV/EBITDA, P/FCF, DCF, EPV, NAV)
  • ✓ สมมติฐานการเติบโตสมเหตุสมผล (15% → 8%)
  • ✓ WACC คำนวณจาก CAPM (9.0%)
  • ✓ Terminal Growth Rate 2.5% (ไม่เกิน GDP Growth)
  • ✓ ทำ Sensitivity Analysis
  • ✓ ผลลัพธ์จากหลายวิธีเปรียบเทียบกัน (Football Field)

✅ หลัง Valuation

  • ☐ Margin of Safety: 4.6% ⚠️ (ต่ำกว่า 20% ที่แนะนำ)
  • ✓ ตรวจสอบ Value Trap: ROE ต่ำ + รายได้ผันผวน → มีความเสี่ยง
  • ✓ ปัจจัยเชิงคุณภาพ: ธุรกิจขึ้นกับ Backlog, แข่งขันสูง
  • ☐ ตรวจสอบสัญญาณปั่นหุ้น (ตามบทที่ 1-4)
  • ☐ บันทึกใน Trading Journal
