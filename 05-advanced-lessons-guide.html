ภาค 5 — Advanced Lessons

บทเรียนขั้นสูง
จากเอาตัวรอดสู่เติบโตอย่างยั่งยืน

Market Microstructure, Intermarket Analysis, Quantitative Thinking, Behavioral Finance และอีกมากมาย — ทุกสิ่งที่นักลงทุนขั้นสูงต้องรู้

20 Market Microstructure — เข้าใจกลไกตลาดในระดับลึก

Order Book Dynamics — อ่าน Bid-Offer ให้เป็นมากกว่าแค่ดูตัวเลข

นักลงทุนส่วนใหญ่ดู Bid-Offer แค่ "ใครเยอะกว่า" แต่นักลงทุนขั้นสูงอ่าน Order Book เหมือนอ่านภาษา — มันบอกเจตนาของผู้เล่นแต่ละฝ่าย

Order Flow Analysis — อ่านกระแสคำสั่ง

CONCEPT ประเภทคำสั่งและความหมาย: Market Order (คำสั่งตลาด): ├── ซื้อ Market Order = ต้องการหุ้นเดี๋ยวนี้ ยอมจ่ายราคา Offer │ → แสดงความเร่งด่วน = แรงซื้อจริง └── ขาย Market Order = ต้องการเงินเดี๋ยวนี้ ยอมขายราคา Bid → แสดงความเร่งด่วน = แรงขายจริง Limit Order (คำสั่งจำกัดราคา): ├── Bid (ตั้งซื้อ) = "ผมยินดีซื้อที่ราคานี้ ถ้าราคาลงมา" │ → แสดงความต้องการ แต่ไม่เร่งด่วน └── Offer (ตั้งขาย) = "ผมยินดีขายที่ราคานี้ ถ้าราคาขึ้นไป" → แสดงความต้องการขาย แต่ไม่เร่งด่วน

วิธีอ่าน Order Flow

📊 สัญญาณจาก Order Flow
  • • Market Buy > Market Sell → แรงซื้อจริงแข็งแกร่ง
  • • Bid ถูก Hit บ่อยแต่ราคาไม่ลง → มีคนรอรับซื้อด้านล่างตลอด = สัญญาณสะสม (Accumulation)
  • • Offer ถูก Lift บ่อยแต่ราคาไม่ขึ้น → มีคนเติม Offer ใหม่ตลอด = สัญญาณกระจายของ (Distribution)

Spoofing และ Layering — กลลวงใน Order Book

ผู้ปั่นไม่ได้แค่ซื้อขายจริง พวกเขายังใช้ Order Book เป็นเครื่องมือหลอกลวง:

SPOOFING สถานการณ์: ผู้ปั่นต้องการซื้อหุ้นในราคาถูก ขั้นตอน: 1. วาง Offer ขนาดใหญ่ที่ราคา 10.10 (500,000 หุ้น) → คนเห็นว่ามีแรงขายหนัก → กลัว → ขายหุ้นออกมา 2. รับซื้อหุ้นที่คนขายออกมาที่ราคา 9.90-10.00 3. ดึง Offer 500,000 หุ้นออก (ไม่เคยตั้งใจจะขายจริง) 4. ราคาวิ่งขึ้นเพราะ "กำแพงขาย" หายไป ผลลัพธ์: ซื้อหุ้นได้ในราคาถูกกว่าที่ควรจะเป็น
LAYERING สถานการณ์: ผู้ปั่นต้องการดันราคาขึ้น ขั้นตอน: 1. วาง Bid หลายชั้นด้านล่าง: - 100,000 หุ้น ที่ 10.00 - 200,000 หุ้น ที่ 9.90 - 300,000 หุ้น ที่ 9.80 → ดูเหมือนมีแรงซื้อรอรับหนามาก 2. คนเห็นว่า "มีคนรอซื้อเยอะ" → มั่นใจ → ซื้อตาม 3. ราคาขึ้น → ผู้ปั่นขายของออก 4. ดึง Bid ทั้งหมดออก ผลลัพธ์: สร้างภาพลวงว่ามีแรงซื้อหนัก ทั้งที่ไม่มี
⚠️ วิธีจับ Spoofing/Layering
  • Bid/Offer ขนาดใหญ่วางแล้วดึงออกภายใน 1-5 นาที → น่าสงสัย
  • Bid/Offer ขนาดใหญ่ถูกดึงออกเมื่อราคาใกล้ถึง → Spoofing
  • รูปแบบซ้ำๆ ในวันเดียว — Bid/Offer ใหญ่ปรากฏและหายไปซ้ำๆ → มีคนเล่นเกม

Iceberg Orders — ออเดอร์ที่ซ่อนอยู่ใต้น้ำ

🧊 Iceberg Order คืออะไร

คำสั่งซื้อ/ขายขนาดใหญ่ที่แสดงบน Order Book แค่ส่วนเล็กๆ เหมือนภูเขาน้ำแข็งที่โผล่พ้นน้ำแค่ยอด

EXAMPLE ตัวอย่าง: คำสั่งซื้อจริง: 1,000,000 หุ้น แสดงบน Bid: 10,000 หุ้น ทุกครั้งที่ 10,000 หุ้นถูก Match → ระบบเติม 10,000 หุ้นใหม่อัตโนมัติ ดูจากหน้าจอเหมือนมีคนซื้อแค่ 10,000 หุ้น แต่จริงๆ มีคนรอซื้อ 1,000,000 หุ้น
💡 วิธีจับ Iceberg Order
  • Bid/Offer ที่ช่องราคาเดียวกัน "เติม" ซ้ำๆ หลังถูก Match
  • Time & Sales แสดงรายการซื้อขายที่ราคาเดียวกันซ้ำๆ หลายสิบครั้ง
  • วอลุ่มสะสมสูงมากแต่ Bid/Offer ที่แสดงน้อย
📘 ข้อสังเกต
Iceberg Order ไม่ได้ผิดกฎหมาย — เป็นเครื่องมือที่สถาบันใช้เพื่อไม่ให้ตลาดรู้ว่ากำลังซื้อ/ขายจำนวนมาก ถ้าคุณจับได้ว่ามี Iceberg Buy → แสดงว่ามีรายใหญ่กำลังสะสม = ข้อมูลที่มีค่ามาก

Tick Data Analysis — วิเคราะห์ทุกจังหวะการซื้อขาย

Tick Data คือข้อมูลการซื้อขายทุกรายการ (ทุก Tick) ไม่ใช่แค่ราคาเปิด-ปิด-สูง-ต่ำ

🔵 Aggressor Analysis

  • Trades ที่ราคา Offer = Buy Aggressor = แรงซื้อเป็นฝ่ายรุก
  • Trades ที่ราคา Bid = Sell Aggressor = แรงขายเป็นฝ่ายรุก

🟢 Trade Size Distribution

  • รายการเล็ก (<5,000 หุ้น) = รายย่อย
  • รายการใหญ่ (>50,000 หุ้น) = สถาบัน
  • รายการขนาดเท่าๆ กันซ้ำๆ = Algorithm/Iceberg

🟣 Time of Trade

  • 10:00-10:30 = ดันราคาตอนเปิด (Opening Push)
  • 16:00-16:30 = Marking the Close
  • 11:00-14:00 = การซื้อขายจริงของสถาบัน

21 Intermarket Analysis — อ่านความสัมพันธ์ระหว่างตลาด

ตลาดหุ้นไม่ได้อยู่โดดเดี่ยว มันเชื่อมโยงกับตลาดอื่นๆ อย่างแน่นแฟ้น นักลงทุนขั้นสูงต้องเข้าใจความสัมพันธ์เหล่านี้เพื่อ "เห็นภาพใหญ่" ก่อนตัดสินใจ

ความสัมพันธ์หลัก 4 ตลาด

🌐 4 ตลาดที่เชื่อมโยงกัน
CONCEPT ┌──────────────────┐ ┌──────────────────┐ │ ตลาดพันธบัตร │ ←→ │ ตลาดหุ้น │ │ (Bond Market) │ │ (Stock Market) │ └────────┬─────────┘ └────────┬─────────┘ │ │ ▼ ▼ ┌──────────────────┐ ┌──────────────────┐ │ ตลาดค่าเงิน │ ←→ │ ตลาดสินค้าโภค │ │ (Currency Mkt) │ │ (Commodity Mkt) │ └──────────────────┘ └──────────────────┘

พันธบัตร vs หุ้น

BOND YIELD UP Bond Yield ขึ้น (ดอกเบี้ยสูงขึ้น) ├── เงินไหลจากหุ้นไปพันธบัตร (ผลตอบแทนดีกว่า ปลอดภัยกว่า) ├── ต้นทุนการกู้ยืมของบริษัทสูงขึ้น → กำไรลดลง ├── Discount Rate สูงขึ้น → มูลค่าปัจจุบันของกำไรในอนาคตลดลง └── กดดันหุ้น Growth (P/E สูง) มากกว่าหุ้น Value
BOND YIELD DOWN Bond Yield ลง (ดอกเบี้ยต่ำลง) ├── เงินไหลจากพันธบัตรไปหุ้น (หาผลตอบแทนที่สูงกว่า) ├── ต้นทุนการกู้ยืมต่ำ → บริษัทขยายธุรกิจง่ายขึ้น └── หนุนหุ้น Growth มากกว่าหุ้น Value
📌 ตัวชี้วัดที่ต้องดู
  • • US 10-Year Treasury Yield — ดอกเบี้ยพันธบัตรสหรัฐ 10 ปี (ตัวชี้วัดสำคัญที่สุดของโลก)
  • • Thai Government Bond Yield — ดอกเบี้ยพันธบัตรรัฐบาลไทย
  • • Yield Curve — ถ้า Inverted (ดอกเบี้ยระยะสั้น > ระยะยาว) → สัญญาณเตือนเศรษฐกิจถดถอย

ค่าเงินบาท vs หุ้นไทย

STRONG BAHT บาทแข็ง (USD/THB ลดลง): ├── ดีกับ: หุ้นนำเข้า (ต้นทุนวัตถุดิบถูกลง) ├── แย่กับ: หุ้นส่งออก (รายได้เป็นเงินบาทน้อยลง) ├── ดีกับ: หุ้นท่องเที่ยว (ต่างชาติมาเที่ยวถูกลง) └── สัญญาณ: เงินทุนต่างชาติไหลเข้า → หนุนตลาดหุ้น
WEAK BAHT บาทอ่อน (USD/THB เพิ่มขึ้น): ├── ดีกับ: หุ้นส่งออก (รายได้เป็นเงินบาทมากขึ้น) ├── แย่กับ: หุ้นนำเข้า (ต้นทุนวัตถุดิบแพงขึ้น) ├── แย่กับ: บริษัทที่มีหนี้เป็นเงินดอลลาร์ └── สัญญาณ: เงินทุนต่างชาติไหลออก → กดดันตลาดหุ้น

น้ำมัน vs หุ้นไทย

OIL UP น้ำมันขึ้น: ├── ดีกับ: PTT, PTTEP, BANPU (หุ้นพลังงาน) ├── แย่กับ: สายการบิน (ต้นทุนน้ำมันสูงขึ้น) ├── แย่กับ: หุ้นขนส่ง (ต้นทุนเชื้อเพลิงสูงขึ้น) └── กดดัน: เงินเฟ้อสูงขึ้น → ดอกเบี้ยอาจขึ้น → กดดันตลาดรวม
OIL DOWN น้ำมันลง: ├── แย่กับ: หุ้นพลังงาน ├── ดีกับ: สายการบิน, ขนส่ง └── หนุน: เงินเฟ้อลดลง → ดอกเบี้ยอาจลง → หนุนตลาดรวม

ทองคำ — ตัวชี้วัดความกลัว

GOLD UP ทองคำขึ้น: ├── สัญญาณ: นักลงทุนกลัว (Risk-off) ├── มักเกิดพร้อมกับ: หุ้นลง, Bond Yield ลง, ดอลลาร์อ่อน └── ความหมาย: ตลาดกำลังหนีไปหาสินทรัพย์ปลอดภัย
GOLD DOWN ทองคำลง: ├── สัญญาณ: นักลงทุนมั่นใจ (Risk-on) ├── มักเกิดพร้อมกับ: หุ้นขึ้น, Bond Yield ขึ้น, ดอลลาร์แข็ง └── ความหมาย: ตลาดกำลังเสี่ยงมากขึ้น

Fund Flow Analysis — ติดตามเงินทุนไหลเข้าออก

DAILY DATA ข้อมูลที่ต้องดูทุกวัน: 1. Net Buy/Sell ของนักลงทุนต่างชาติ - ซื้อสุทธิต่อเนื่อง → เงินไหลเข้า → หนุนตลาด - ขายสุทธิต่อเนื่อง → เงินไหลออก → กดดันตลาด 2. Net Buy/Sell ของนักลงทุนสถาบัน - สถาบันมักเป็น "Smart Money" — ดูว่าพวกเขาซื้อหรือขาย 3. Net Buy/Sell ของนักลงทุนรายย่อย - รายย่อยมักเป็น Contrarian Indicator - ถ้ารายย่อยซื้อหนักมาก → ระวัง ตลาดอาจใกล้จุดสูงสุด - ถ้ารายย่อยขายหนักมาก → อาจเป็นโอกาสซื้อ
💡 สัญญาณบวกที่แข็งแกร่งที่สุด
ต่างชาติซื้อ + สถาบันซื้อ + รายย่อยขาย → Best Signal
🚨 สัญญาณลบที่อันตรายที่สุด
ต่างชาติขาย + สถาบันขาย + รายย่อยซื้อ → Worst Signal

22 Quantitative Thinking — คิดเป็นตัวเลข ไม่ใช่ความรู้สึก

Expectancy — ค่าคาดหวังของระบบ

นักลงทุนขั้นสูงไม่ได้ถามว่า "เทรดนี้จะกำไรไหม?" แต่ถามว่า "ระบบของผมมี Expectancy เป็นบวกไหม?"

FORMULA Expectancy = (Win Rate × Avg Win) - (Loss Rate × Avg Loss)
SYSTEM A ตัวอย่างระบบ A: Win Rate = 45% Average Win = 25,000 บาท Loss Rate = 55% Average Loss = 12,000 บาท Expectancy = (0.45 × 25,000) - (0.55 × 12,000) = 11,250 - 6,600 = +4,650 บาทต่อเทรด ✅
SYSTEM B ตัวอย่างระบบ B: Win Rate = 70% Average Win = 8,000 บาท Loss Rate = 30% Average Loss = 25,000 บาท Expectancy = (0.70 × 8,000) - (0.30 × 25,000) = 5,600 - 7,500 = -1,900 บาทต่อเทรด ❌
⚠️ ข้อสังเกตสำคัญ
ระบบ A ถูกแค่ 45% แต่ทำเงินได้ ระบบ B ถูกถึง 70% แต่ขาดทุน — Win Rate สูงไม่ได้แปลว่าดีเสมอไป สิ่งที่สำคัญคืออัตราส่วนระหว่างกำไรเฉลี่ยกับขาดทุนเฉลี่ย

R-Multiple — วัดผลเทรดด้วยหน่วย "R"

R คือ "ความเสี่ยงเริ่มต้น" ของแต่ละเทรด ใช้เป็นหน่วยวัดมาตรฐานเพื่อเปรียบเทียบเทรดต่างๆ ได้

EXAMPLE R = (ราคาซื้อ - Stop Loss) × จำนวนหุ้น ตัวอย่าง: ซื้อ 10,000 หุ้น ที่ 10.00 บาท Stop Loss ที่ 9.00 บาท R = (10.00 - 9.00) × 10,000 = 10,000 บาท ถ้าขายได้กำไรที่ 12.00 บาท: กำไร = (12.00 - 10.00) × 10,000 = 20,000 บาท R-Multiple = 20,000 / 10,000 = +2R ✅ ถ้าขายขาดทุนที่ 9.00 บาท: ขาดทุน = (10.00 - 9.00) × 10,000 = 10,000 บาท R-Multiple = -10,000 / 10,000 = -1R ❌
📐 ทำไม R-Multiple สำคัญ
  • • ทำให้เปรียบเทียบเทรดที่ Position Size ต่างกันได้
  • • เป้าหมาย: เทรดที่กำไรควรได้ +2R ขึ้นไป เทรดที่ขาดทุนควรไม่เกิน -1R
  • • ดู Distribution ของ R-Multiple ย้อนหลัง 50-100 เทรด จะเห็นภาพชัดว่าระบบทำงานอย่างไร

Monte Carlo Simulation — ทดสอบระบบด้วยการจำลอง

STEPS วิธีทำ Monte Carlo แบบง่าย: 1. เก็บผลลัพธ์ R-Multiple ของเทรดย้อนหลัง 50-100 ครั้ง เช่น: +2R, -1R, +1.5R, -1R, +3R, -1R, +0.5R, ... 2. สุ่มลำดับใหม่ 1,000 ครั้ง (ใช้ Excel หรือ Python) → แต่ละครั้งจะได้ Equity Curve ที่ต่างกัน 3. ดูผลลัพธ์: - Worst Case Drawdown คือเท่าไหร่? - มีโอกาสกี่ % ที่จะขาดทุนติดต่อกัน 10 ครั้ง? - ต้องมีเงินทุนเท่าไหร่ถึงจะรอดจาก Worst Case? 4. ปรับ Position Size ให้รอดจาก Worst Case ได้
💡 กฎง่ายๆ
ถ้า Monte Carlo บอกว่า Worst Case Drawdown คือ -40% แต่คุณรับได้แค่ -20% → ต้องลด Position Size ลงครึ่งหนึ่ง ดีกว่าเจอ Drawdown แล้วหมดกำลังใจเลิกเทรด

Kelly Criterion — Position Sizing ที่เหมาะสมทางคณิตศาสตร์

FORMULA Kelly % = W - [(1 - W) / R] W = Win Rate (อัตราชนะ) R = Win/Loss Ratio (กำไรเฉลี่ย / ขาดทุนเฉลี่ย) ตัวอย่าง: Win Rate = 45% (W = 0.45) Average Win = 25,000 บาท Average Loss = 12,000 บาท R = 25,000 / 12,000 = 2.08 Kelly % = 0.45 - [(1 - 0.45) / 2.08] = 0.45 - [0.55 / 2.08] = 0.45 - 0.264 = 0.186 = 18.6% → ควรลงทุน 18.6% ของพอร์ตต่อเทรด (ตาม Full Kelly)
⚠️ Never use Full Kelly
Full Kelly ให้ผลตอบแทนสูงสุดทางทฤษฎี แต่ Drawdown จะรุนแรงมาก ในทางปฏิบัติ:
  • Full Kelly = 18.6% → Drawdown อาจถึง 50-60%
  • Half Kelly = 9.3% → Drawdown ลดเหลือ 25-30%
  • Quarter Kelly = 4.65% → Drawdown ลดเหลือ 12-15%

23 Advanced Technical Analysis — เทคนิคขั้นสูง

Volume Profile — แนวรับแนวต้านที่แท้จริง

Volume Profile แสดงปริมาณการซื้อขายที่แต่ละระดับราคา (ไม่ใช่ตามเวลา) ทำให้เห็นว่า "ราคาไหนมีคนซื้อขายเยอะที่สุด"

VOLUME PROFILE ราคา │ │ ████████████████ ← High Volume Node (HVN) │ ████████████████████ มีคนซื้อขายเยอะ = แนวรับ/ต้านแข็งแกร่ง │ ████████████████ │ ██████ ← Low Volume Node (LVN) │ ████ มีคนซื้อขายน้อย = ราคาวิ่งผ่านเร็ว │ ██████ │ ████████████████████ ← Point of Control (POC) │ ██████████████████████ ราคาที่มีวอลุ่มสูงที่สุด │ ████████████████████ │ ┊← Value Area (70%) →┊ └──────────────────────── วอลุ่ม
📊 แนวคิดหลัก
  • • POC (Point of Control): ราคาที่มีวอลุ่มสูงที่สุด = "ราคายุติธรรม" ที่ตลาดยอมรับ
  • • HVN (High Volume Node): ราคาที่มีการซื้อขายหนัก = แนวรับ/ต้านที่แข็งแกร่ง
  • • LVN (Low Volume Node): ราคาที่มีการซื้อขายน้อย = ราคาจะวิ่งผ่านโซนนี้เร็ว
  • • Value Area: ช่วงราคาที่มีวอลุ่ม 70% ของทั้งหมด = "โซนที่ตลาดยอมรับ"

วิธีใช้ Volume Profile

  • 1. หาแนวรับ-แนวต้านจาก HVN — แม่นยำกว่าการลากเส้นแนวนอนแบบเดิม
  • 2. หาจุดที่ราคาจะวิ่งเร็วจาก LVN — เมื่อราคาทะลุ HVN เข้าสู่ LVN จะวิ่งเร็วจนถึง HVN ถัดไป
  • 3. ดู POC เปลี่ยนแปลงอย่างไร — ถ้า POC ค่อยๆ ขยับขึ้น = เทรนด์ขาขึ้น

VWAP — ราคาเฉลี่ยถ่วงน้ำหนักด้วยวอลุ่ม

FORMULA VWAP = Σ(ราคา × วอลุ่ม) / Σ(วอลุ่ม)
📈 ทำไม VWAP สำคัญ
  • • สถาบันใช้ VWAP เป็น Benchmark — ถ้าซื้อได้ต่ำกว่า VWAP ถือว่า "ซื้อได้ดี"
  • • VWAP ทำหน้าที่เป็นแนวรับ/ต้านแบบ Dynamic
  • • ราคาเหนือ VWAP → ผู้ซื้อส่วนใหญ่กำไร → Sentiment เป็นบวก
  • • ราคาต่ำกว่า VWAP → ผู้ซื้อส่วนใหญ่ขาดทุน → Sentiment เป็นลบ

วิธีใช้ VWAP

  • • ซื้อเมื่อราคา Pullback มาแตะ VWAP แล้วเด้งขึ้น (ในวันที่เทรนด์ขึ้น)
  • • ขายเมื่อราคาดีดขึ้นมาแตะ VWAP แล้วกลับลง (ในวันที่เทรนด์ลง)
  • • ใช้ VWAP ± 1 Standard Deviation เป็นแนวรับ/ต้านเพิ่มเติม

Market Profile — อ่านโครงสร้างตลาดรายวัน

Market Profile แสดงการกระจายตัวของราคาตามเวลา ช่วยให้เห็นว่าตลาดอยู่ในสถานะไหน

NORMAL DAY 1. Normal Day (วันปกติ): ราคา │ A │ AB │ ABC │ ABCD ← รูปทรงระฆัง │ ABCDE ตลาดสมดุล │ ABCD │ ABC │ AB │ A วิธีใช้: เทรดในกรอบ Value Area ซื้อที่ขอบล่าง ขายที่ขอบบน
TREND DAY 2. Trend Day (วันเทรนด์): ราคา │ A │ AB │ BC │ CD ← ราคาเคลื่อนไหวทิศทางเดียว │ DE ตลาดไม่สมดุล │ EF โอกาสทำกำไรสูง │ FG │ GH วิธีใช้: เทรดตามเทรนด์ อย่าสวนทาง ใช้ Pullback เป็นจุดเข้า
DOUBLE DIST 3. Double Distribution (สองยอด): ราคา │ AB │ ABC │ AB ← มีสองโซนที่ซื้อขายหนัก │ A ตลาดกำลังหาจุดสมดุลใหม่ │ AB │ ABC │ AB วิธีใช้: รอดูว่าราคาจะเลือกอยู่โซนไหน แล้วเทรดตาม

Relative Strength — เปรียบเทียบความแข็งแกร่ง

FORMULA RS = ราคาหุ้น / ราคาดัชนี (หรือหุ้นอ้างอิง) ถ้า RS ขึ้น → หุ้นแข็งแกร่งกว่าตลาด (Outperform) ถ้า RS ลง → หุ้นอ่อนแอกว่าตลาด (Underperform)
📊 วิธีใช้ Relative Strength (3 วิธี)
  • 1. เลือกหุ้นที่แข็งแกร่งกว่าตลาด — หุ้นที่ RS ขึ้นมักจะขึ้นต่อ (Momentum Effect)
  • 2. หลีกเลี่ยงหุ้นที่อ่อนแอกว่าตลาด — แม้จะดู "ถูก" แต่ถ้า RS ลง แสดงว่ามีปัญหาบางอย่าง
  • 3. จับจังหวะ Sector Rotation — เมื่อ RS ของกลุ่มอุตสาหกรรมเริ่มเปลี่ยนทิศ → เงินกำลังไหลเข้า/ออกจากกลุ่มนั้น

24 Regime Detection — อ่านสถานะตลาดให้ออก

ตลาดมี 4 สถานะ (Market Regimes)

ตลาดไม่ได้มีแค่ "ขึ้น" กับ "ลง" แต่มี 4 สถานะที่ต้องใช้กลยุทธ์ต่างกัน:

4 REGIMES Volatility ต่ำ Volatility สูง ┌──────────────────┬──────────────────┐ เทรนด์ขึ้น │ Quiet Bull │ Volatile Bull │ │ ขึ้นช้าๆ สม่ำเสมอ │ ขึ้นแรง แกว่งแรง │ │ → ถือยาว │ → เทรดสั้น ตั้ง SL │ ├──────────────────┼──────────────────┤ เทรนด์ลง │ Quiet Bear │ Volatile Bear │ │ ลงช้าๆ ค่อยๆ ไหล │ ลงแรง ตื่นตระหนก │ │ → ถือเงินสด │ → ห้ามซื้อ รอ │ └──────────────────┴──────────────────┘
🔍 วิธีระบุ Regime
  • 1. ทิศทาง (Direction): ดู MA50 vs MA200
  • - MA50 > MA200 → เทรนด์ขึ้น
  • - MA50 < MA200 → เทรนด์ลง
  • 2. ความผันผวน (Volatility): ดู ATR หรือ Bollinger Band Width
  • - ATR ต่ำกว่าค่าเฉลี่ย → Volatility ต่ำ
  • - ATR สูงกว่าค่าเฉลี่ย → Volatility สูง

Quiet Bearเล็กมากแคบถือเงินสดเป็นหลัก, ซื้อเฉพาะหุ้นที่แข็งแกร่งมาก
Volatile Bearไม่ซื้อ—ถือเงินสด 100%, รอจนตลาดนิ่ง

Breadth Indicators — วัดสุขภาพของตลาดโดยรวม

ดัชนี SET อาจขึ้น แต่ถ้าหุ้นส่วนใหญ่ลง แสดงว่าตลาดไม่แข็งแรงจริง Breadth Indicators ช่วยวัดสิ่งนี้

📊 Advance-Decline Line

  • SET ขึ้น + A/D ขึ้น = ขาขึ้นแข็งแกร่ง
  • SET ขึ้น + A/D ลง = ⚠️ ขาขึ้นอ่อนแอ (แค่หุ้นใหญ่ไม่กี่ตัวดัน)
  • SET ลง + A/D ขึ้น = อาจใกล้กลับตัวขึ้น
  • SET ลง + A/D ลง = ขาลงแข็งแกร่ง

📈 New High - New Low

  • New High มาก + New Low น้อย = ตลาดแข็งแกร่ง
  • New High น้อย + New Low มาก = ตลาดอ่อนแอ
  • New High ลดลงขณะ SET ทำ New High = ⚠️ Divergence อันตราย

📉 % Stocks Above MA200

  • >70% = ร้อนแรง อาจ Overbought
  • 50-70% = แข็งแกร่ง
  • 30-50% = ปานกลาง
  • <30% = อ่อนแอ อาจ Oversold

25 Advanced Risk Management — บริหารความเสี่ยงระดับมืออาชีพ

Correlation Risk — ความเสี่ยงที่มองไม่เห็น

ในภาวะปกติ หุ้นแต่ละตัวอาจเคลื่อนไหวต่างทิศทาง แต่ในภาวะวิกฤต Correlation พุ่งขึ้นเข้าหา 1 — หุ้นทุกตัวลงพร้อมกัน

CORRELATION ภาวะปกติ: หุ้น A: ↑ หุ้น B: ↓ หุ้น C: ↑ หุ้น D: → หุ้น E: ↓ → พอร์ตสมดุล กำไร-ขาดทุนหักล้างกัน ภาวะวิกฤต: หุ้น A: ↓↓ หุ้น B: ↓↓ หุ้น C: ↓↓ หุ้น D: ↓↓ หุ้น E: ↓↓ → ทุกตัวลงพร้อมกัน การกระจายไม่ช่วย
🛡️ 4 วิธีจัดการ Correlation Risk
  • 1. ถือเงินสดเป็นส่วนหนึ่งของพอร์ตเสมอ — เงินสดมี Correlation เป็น 0 กับหุ้น
  • 2. ลด Position Size รวมเมื่อ Correlation สูงขึ้น — วัดจาก VIX หรือ Implied Volatility
  • 3. ใช้ Stress Test — ถามว่า "ถ้าหุ้นทุกตัวลง 20% พร้อมกัน พอร์ตจะเป็นอย่างไร?"
  • 4. กระจายข้ามสินทรัพย์ — ไม่ใช่แค่กระจายหุ้น แต่กระจายไปทองคำ พันธบัตร เงินสด

Value at Risk (VaR) — วัดความเสี่ยงเป็นตัวเลข

FORMULA VaR = พอร์ต × Z-score × Daily Standard Deviation สำหรับ 95% confidence: Z-score = 1.65 สำหรับ 99% confidence: Z-score = 2.33 ตัวอย่าง: พอร์ต = 1,000,000 บาท SD รายวันของพอร์ต = 1.5% VaR 95% = 1,000,000 × 1.65 × 0.015 = 24,750 บาท ความหมาย: ใน 95% ของวัน พอร์ตจะขาดทุนไม่เกิน 24,750 บาท แต่ใน 5% ของวัน (ประมาณ 1 วันต่อเดือน) อาจขาดทุนมากกว่านั้น
⚠️ ข้อจำกัดของ VaR
  • VaR บอกแค่ "ขาดทุนไม่เกินเท่าไหร่ในกรณีปกติ" แต่ไม่บอกว่า "ถ้าเกินจะเป็นเท่าไหร่"
  • ในวิกฤต (Tail Risk) ขาดทุนจริงอาจมากกว่า VaR หลายเท่า
  • ใช้ VaR เป็น "เครื่องมือตรวจสอบ" ไม่ใช่ "เครื่องมือตัดสินใจ"

Maximum Drawdown Management — จัดการช่วงขาดทุนสะสม

DRAWDOWN RULES กฎ Drawdown Management: ├── Drawdown 0-10%: ปกติ เทรดตามระบบ ├── Drawdown 10-15%: ลด Position Size ลง 25% ├── Drawdown 15-20%: ลด Position Size ลง 50% ├── Drawdown 20-25%: ลด Position Size ลง 75% └── Drawdown >25%: หยุดเทรด ทบทวนระบบทั้งหมด

Drawdown ผลตอบแทนที่ต้องทำ เวลาฟื้นตัว (สมมติทำได้ 15%/ปี)
-10%+11.1%~9 เดือน
-20%+25.0%~1.7 ปี
-30%+42.9%~2.6 ปี

26 Behavioral Finance ขั้นสูง — เข้าใจสมองเพื่อเอาชนะตัวเอง

Prospect Theory — ทำไมคนตัดสินใจผิดซ้ำแล้วซ้ำเล่า

EXPERIMENT การทดลองคลาสสิก: ตัวเลือก A: ได้เงิน 100,000 บาทแน่นอน ตัวเลือก B: มีโอกาส 50% ได้ 200,000 บาท หรือ 50% ไม่ได้อะไร Expected Value เท่ากัน (100,000 บาท) แต่คนส่วนใหญ่เลือก A → เมื่อเป็นเรื่องกำไร คนเลือกความแน่นอน (Risk Averse) ตัวเลือก C: เสียเงิน 100,000 บาทแน่นอน ตัวเลือก D: มีโอกาส 50% เสีย 200,000 บาท หรือ 50% ไม่เสียอะไร Expected Value เท่ากัน (-100,000 บาท) แต่คนส่วนใหญ่เลือก D → เมื่อเป็นเรื่องขาดทุน คนเลือกเสี่ยง (Risk Seeking)
💡 ผลกระทบต่อการลงทุน
  • • ตอนกำไร → รีบขาย (กลัวกำไรหาย) → ตัดกำไรเร็วเกินไป
  • • ตอนขาดทุน → ถือต่อ (หวังว่าจะกลับมา) → ปล่อยขาดทุนวิ่ง

3 วิธีต่อสู้กับ Prospect Theory

  • 1. ตั้งกฎล่วงหน้า — ตัดสินใจตอนที่ยังไม่มีอารมณ์เข้ามาเกี่ยว
  • 2. คิดเป็น R-Multiple — แทนที่จะคิดว่า "กำไร 50,000 บาท" ให้คิดว่า "+2R" ทำให้ตัดอารมณ์ออก
  • 3. ใช้ระบบอัตโนมัติ — ตั้ง Trailing Stop ให้ระบบจัดการ ไม่ต้องตัดสินใจเอง

System 1 vs System 2 — สมองสองระบบ

⚡ System 1 (สมองเร็ว)

  • ทำงานอัตโนมัติ ไม่ต้องคิด
  • ใช้สัญชาตญาณ อารมณ์ ประสบการณ์
  • เร็วมาก แต่มักผิดพลาดในสถานการณ์ซับซ้อน
  • ตัวอย่าง: เห็นหุ้นวิ่ง → กลัวตกรถ → กดซื้อทันที

🧠 System 2 (สมองช้า)

  • ต้องใช้ความพยายามในการคิด
  • ใช้เหตุผล ตรรกะ การวิเคราะห์
  • ช้า แต่แม่นยำกว่าในสถานการณ์ซับซ้อน
  • ตัวอย่าง: คำนวณ Position Size → ตรวจสอบ R:R → ตั้ง Stop Loss → ค่อยซื้อ
⚠️ ปัญหา: System 1 มักชนะ System 2 เพราะ
  • • ราคาเคลื่อนไหวเร็ว → สมองรู้สึกว่าต้อง "รีบตัดสินใจ"
  • • อารมณ์ (กลัว โลภ) กระตุ้น System 1 ให้ทำงาน
  • • System 2 ต้องใช้พลังงาน → เมื่อเหนื่อยหรือเครียด System 2 จะหยุดทำงาน

4 วิธีบังคับให้ System 2 ทำงาน

  • 1. Checklist — บังคับให้ผ่านขั้นตอนก่อนซื้อ/ขาย
  • 2. Cooling Period — รอ 30 นาทีหลังเห็นสัญญาณก่อนตัดสินใจ
  • 3. เขียนเหตุผล — ถ้าเขียนไม่ได้ แปลว่ายังคิดไม่รอบคอบ
  • 4. พักผ่อนให้พอ — System 2 ต้องการพลังงาน นอนน้อย = ตัดสินใจแย่

Narrative Fallacy — กับดักของ "เรื่องเล่า"

มนุษย์ชอบ "เรื่องเล่า" มากกว่า "ตัวเลข" สมองถูกออกแบบมาให้จดจำเรื่องราวได้ดีกว่าสถิติ ผู้ปั่นรู้ดีและใช้สิ่งนี้เป็นอาวุธ

CONTRAST เรื่องเล่า: "บริษัท XYZ กำลังจะได้สัมปทานพลังงานสะอาดมูลค่า 10,000 ล้าน CEO เป็นอดีตผู้บริหาร PTT มีคอนเนคชั่นระดับรัฐมนตรี หุ้นตัวนี้จะเป็น 10-Bagger ภายใน 2 ปี" ตัวเลข: Market Cap 800 ล้าน, รายได้ปีล่าสุด 50 ล้าน, ขาดทุนสุทธิ 30 ล้าน P/E ติดลบ, D/E Ratio 3.5, Operating CF ติดลบ 3 ปีติด ผู้บริหารเคยถูก ก.ล.ต. ตักเตือน
🚨 คนส่วนใหญ่จำ "เรื่องเล่า" ได้ แต่ไม่ได้ดู "ตัวเลข"
🛡️ 4 วิธีต่อสู้กับ Narrative Fallacy
  • 1. ดูตัวเลขก่อนเรื่องเล่าเสมอ — อ่านงบการเงินก่อนอ่านข่าว
  • 2. ถามว่า "ตัวเลขสนับสนุนเรื่องเล่าไหม?" — ถ้าไม่ → เรื่องเล่าไม่น่าเชื่อถือ
  • 3. หา "เรื่องเล่าฝั่งตรงข้าม" — ถ้าหาเหตุผลที่หุ้นจะลงไม่ได้ → คุณกำลังมี Confirmation Bias
  • 4. Base Rate Thinking — ถามว่า "จากสถิติ หุ้นที่มีลักษณะแบบนี้ กี่ % ที่สำเร็จจริง?"

27 Building Your Edge — สร้างความได้เปรียบของตัวเอง

Edge คืออะไร

Edge คือ "ความได้เปรียบ" ที่ทำให้คุณมีโอกาสทำกำไรมากกว่าขาดทุนในระยะยาว ถ้าไม่มี Edge คุณกำลังเล่นพนัน ไม่ใช่ลงทุน

📘 1. Informational Edge

  • รู้ข้อมูลที่คนอื่นไม่รู้ (ถูกกฎหมาย)
  • วิเคราะห์งบการเงินลึกกว่าคนอื่น
  • เข้าใจอุตสาหกรรมเฉพาะทาง
  • ยากขึ้นเรื่อยๆ ในยุคที่ข้อมูลเข้าถึงง่าย

📗 2. Analytical Edge

  • วิเคราะห์ข้อมูลเดียวกันได้ดีกว่าคนอื่น
  • เห็น Pattern ที่คนอื่นไม่เห็น
  • เข้าใจ Intermarket Relationship ดีกว่า
  • ต้องฝึกฝนและพัฒนาตลอดเวลา

📕 3. Behavioral Edge

  • ควบคุมอารมณ์ได้ดีกว่าคนอื่น
  • มีวินัยในการทำตามระบบ
  • ไม่ตื่นตระหนกเมื่อตลาดผันผวน
  • Edge ที่ยั่งยืนที่สุด

📙 4. Structural Edge

  • ได้เปรียบจากโครงสร้างตลาด
  • รายย่อยซื้อหุ้นเล็กที่กองทุนใหญ่ซื้อไม่ได้
  • ไม่มีแรงกดดันจาก Benchmark
  • ไม่ต้องรายงานผลทุกไตรมาส

วิธีค้นหา Edge ของตัวเอง

ขั้นตอนที่ 1: ทบทวน Trading Journal 100 เทรดย้อนหลัง

📋 วิเคราะห์
  • • เทรดประเภทไหนที่ทำกำไรบ่อยที่สุด? (Breakout? Pullback? Reversal? Momentum?)
  • • หุ้นกลุ่มไหนที่คุณเข้าใจดีที่สุด?
  • • ช่วงเวลาไหนที่คุณเทรดได้ดีที่สุด?
  • • สถานการณ์ตลาดแบบไหนที่ระบบคุณทำงานดี?
  • • อะไรคือข้อผิดพลาดที่เกิดซ้ำบ่อยที่สุด?

ขั้นตอนที่ 2: สร้าง Playbook ของตัวเอง

SETUP 1 Setup 1: "Breakout จาก Accumulation" ├── เงื่อนไข: Sideway >4 สัปดาห์ + OBV ขึ้น + วอลุ่ม Breakout >3x ├── จุดเข้า: ราคาปิดเหนือแนวต้าน ├── Stop Loss: ใต้กรอบ Sideway ├── เป้าหมาย: ความกว้างของกรอบ Sideway × 1.5 ├── Win Rate ย้อนหลัง: 52% ├── Avg R-Multiple: +1.8R └── Expectancy: +0.54R ต่อเทรด
SETUP 2 Setup 2: "Pullback ในขาขึ้น" ├── เงื่อนไข: ราคาเหนือ MA50 + RSI ลงมาแตะ 40-50 + วอลุ่มลดลง ├── จุดเข้า: แท่งเทียนกลับตัวที่แนวรับ (Hammer, Engulfing) ├── Stop Loss: ใต้ Swing Low ล่าสุด ├── เป้าหมาย: High ก่อนหน้า ├── Win Rate ย้อนหลัง: 58% ├── Avg R-Multiple: +1.5R └── Expectancy: +0.45R ต่อเทรด

ขั้นตอนที่ 3: ทดสอบและปรับปรุง

🔄 กระบวนการที่ไม่มีวันจบ
  • • เทรดตาม Playbook อย่างเคร่งครัด 50 เทรด
  • • บันทึกผลทุกเทรด
  • • วิเคราะห์ว่า Setup ไหนทำงานดี Setup ไหนไม่ดี
  • • ตัด Setup ที่ไม่ดีออก เพิ่ม Setup ใหม่ที่ค้นพบ
  • • ทำซ้ำ — นี่คือกระบวนการที่ไม่มีวันจบ

Compounding — พลังของดอกเบี้ยทบต้น

COMPOUNDING ตัวอย่าง: เริ่มต้น 500,000 บาท ทำได้ 20% ต่อปี ปีที่ 1: 500,000 → 600,000 ปีที่ 3: 500,000 → 864,000 ปีที่ 5: 500,000 → 1,244,160 ปีที่ 10: 500,000 → 3,095,869 ปีที่ 15: 500,000 → 7,703,718 ปีที่ 20: 500,000 → 19,172,783 จาก 500,000 เป็น 19 ล้าน ใน 20 ปี ด้วย 20% ต่อปี ไม่ต้องเสี่ยงมาก ไม่ต้องปั่น ไม่ต้องหาหุ้น 10-Bagger แค่ทำกำไรสม่ำเสมอ + ไม่ขาดทุนหนัก
💡 500k → 19M ใน 20 ปี
ไม่ต้องเสี่ยงมาก ไม่ต้องปั่น ไม่ต้องหาหุ้น 10-Bagger แค่ทำกำไรสม่ำเสมอ + ไม่ขาดทุนหนัก
📐 4 กฎของ Compounding
  • 1. อย่าขาดทุนหนัก — Drawdown ใหญ่ทำลาย Compounding
  • 2. อย่าถอนเงินออกบ่อย — ปล่อยให้เงินทำงาน
  • 3. อดทน — Compounding ใช้เวลา ปีแรกๆ ดูช้า แต่ปีหลังๆ จะเร็วขึ้นแบบทวีคูณ
  • 4. สม่ำเสมอสำคัญกว่าหวือหวา — 20% ต่อปีทุกปี ดีกว่า +100% ปีหนึ่ง -50% อีกปี

28 Meta-Learning — เรียนรู้วิธีเรียนรู้

Deliberate Practice — ฝึกอย่างมีจุดมุ่งหมาย

การเทรดไม่ต่างจากกีฬาหรือดนตรี — ต้องฝึกอย่างมีระบบ ไม่ใช่แค่ "ทำซ้ำ"

📊 1. Chart Replay

  • เปิดกราฟหุ้นที่เคยถูกปั่น ย้อนไปก่อนเริ่มปั่น
  • เลื่อนกราฟทีละวัน ถามตัวเองว่า "จะทำอะไร?"
  • บันทึกการตัดสินใจ แล้วเทียบกับสิ่งที่เกิดขึ้นจริง
  • ทำซ้ำกับหุ้นหลายสิบตัว จนจำ Pattern ได้

📝 2. Paper Trading

  • จำลองพอร์ต 1,000,000 บาท
  • เทรดตามระบบจริง แต่ไม่ใช้เงินจริง
  • บันทึกทุกเทรดเหมือนเทรดจริง
  • ทำอย่างน้อย 50 เทรดก่อนใช้เงินจริง

🧠 3. Mental Rehearsal

  • จินตนาการสถานการณ์ที่ยากลำบาก (หุ้นลง 15% ในวันเดียว)
  • ซ้อมการตัดสินใจในหัวก่อนเจอจริง
  • เมื่อเจอจริง สมองจะพร้อมมากกว่า

After Action Review (AAR) — ทบทวนหลังเทรด

ทุกเทรดคือบทเรียน ไม่ว่าจะกำไรหรือขาดทุน

📋 AAR Template
TEMPLATE 1. เกิดอะไรขึ้น? (What happened?) ซื้อหุ้น ___ ที่ราคา ___ เมื่อวันที่ ___ ขายที่ราคา ___ เมื่อวันที่ ___ ผลลัพธ์: +/-___R 2. ทำไมถึงเป็นแบบนั้น? (Why?) ถ้ากำไร: อะไรทำให้ถูก? ระบบทำงานดีตรงไหน? ถ้าขาดทุน: อะไรผิดพลาด? สัญญาณที่พลาดคืออะไร? 3. ทำตามแผนไหม? (Did I follow the plan?) ✅ ใช่ + กำไร → ระบบดี ทำต่อ ✅ ใช่ + ขาดทุน → ระบบอาจต้องปรับ หรือแค่โชคไม่ดี ❌ ไม่ + กำไร → อันตราย! กำไรจากความโชคดี ไม่ยั่งยืน ❌ ไม่ + ขาดทุน → ต้องแก้ไขวินัย 4. จะทำอะไรต่างไปในครั้งหน้า? ___________________________________

Process vs Outcome — ให้ความสำคัญกับกระบวนการ ไม่ใช่ผลลัพธ์

ผลลัพธ์ดี (กำไร) 📈 ผลลัพธ์แย่ (ขาดทุน) 📉
กระบวนการดี ✅
(ทำตามระบบ)		 ดีที่สุด — ทำต่อไป ยอมรับได้ — ส่วนหนึ่งของเกม
กระบวนการแย่ ❌
(ไม่ทำตามระบบ)		 อันตรายที่สุด — สอนนิสัยที่ผิด สมควรแล้ว — บทเรียนราคาแพง
🚨 เทรดที่อันตรายที่สุด
"กระบวนการแย่ + ผลลัพธ์ดี" เพราะมันสอนให้คุณทำผิดซ้ำ ตัวอย่าง: ไม่ตั้ง Stop Loss แต่บังเอิญกำไร → ครั้งต่อไปก็จะไม่ตั้ง Stop Loss อีก → จนกว่าจะเจอครั้งที่ขาดทุนหนัก

📋 สรุป: 10 หลักการสำคัญสำหรับนักลงทุนขั้นสูง

1
คิดเป็นความน่าจะเป็น ไม่ใช่ความแน่นอน

ไม่มีเทรดไหนที่ "ต้องกำไร" มีแต่เทรดที่ "มีโอกาสกำไรมากกว่าขาดทุน"

2
ปกป้องเงินทุนก่อนหากำไร

Drawdown ทำลาย Compounding ป้องกันขาดทุนหนักสำคัญกว่าไล่ตามกำไรมาก

3
มี Edge ที่วัดได้

ถ้าคุณไม่สามารถอธิบายได้ว่า Edge ของคุณคืออะไร คุณไม่มี Edge

4
ให้ระบบทำงาน ไม่ใช่อารมณ์

ตัดสินใจล่วงหน้า ทำตามแผน ใช้ Checklist

5
เข้าใจ Market Regime

กลยุทธ์ที่ดีในตลาดขาขึ้นอาจแย่ในตลาดขาลง ปรับตัวตามสถานการณ์

6
ดูภาพใหญ่ก่อนภาพเล็ก

Intermarket Analysis, Fund Flow, Breadth Indicators บอกสุขภาพของตลาดโดยรวม

7
อ่าน Order Book ให้เป็น

Spoofing, Iceberg, Layering เป็นเครื่องมือของผู้ปั่น ถ้าอ่านออกจะไม่ตกเป็นเหยื่อ

8
วัดทุกอย่างเป็นตัวเลข

Expectancy, R-Multiple, VaR, Drawdown — ตัวเลขไม่โกหก ความรู้สึกโกหก

9
ฝึกอย่างมีจุดมุ่งหมาย

Chart Replay, Paper Trading, AAR ทำให้คุณดีขึ้นทุกวัน

10
อดทนและสม่ำเสมอ

Compounding ต้องการเวลา ไม่มีทางลัด ความสม่ำเสมอชนะความหวือหวาเสมอ

"ในระยะสั้น ตลาดคือเครื่องลงคะแนนเสียง ในระยะยาว ตลาดคือเครื่องชั่งน้ำหนัก" — Benjamin Graham
